Завершение работы над проектом в IBM Rational Rose 2002

В данной главе описывается процесс завершения разработки проекта в среде IBM Rational Rose 2002. Основное внимание уделяется построению канонических диаграмм и генерации программного кода на языке ANSI C++.

14.1. Разработка диаграммы состояний

Диаграмма состояний используется для описания поведения системы через состояния и переходы между ними. Эта диаграмма может быть построена для отдельного класса, пакета или системы в целом.

Для создания диаграммы состояний в IBM Rational Rose 2002 необходимо выбрать соответствующий инструмент на панели. Основные элементы диаграммы включают состояния, переходы, начальное и конечное состояния. Каждое состояние можно настроить, указав его свойства, такие как действия на входе и выходе, а также внутреннюю деятельность.

Пример диаграммы состояний для системы управления банкоматом включает состояния "Ожидание карточки", "Проверка PIN-кода" и "Выдача наличных". Переходы между состояниями определяются событиями и условиями, такими как вставка карточки или ввод правильного PIN-кода.

14.2. Разработка диаграммы деятельности

Диаграмма деятельности описывает последовательность действий, выполняемых системой. Эта диаграмма может быть построена для отдельного класса, пакета или варианта использования.

Для создания диаграммы деятельности используются элементы, такие как деятельность, переход, ветвление и дорожки. Каждая деятельность может быть настроена с указанием переходов и дорожек, которые определяют поток выполнения.

Пример диаграммы деятельности для варианта использования "Снятие наличных" включает этапы "Вставить карточку", "Ввести PIN-код" и "Выбрать сумму". Переходы между деятельностями определяются условиями и событиями, такими как успешное введение PIN-кода или выбор суммы.

14.3. Разработка диаграммы компонентов

Диаграмма компонентов описывает физическое представление системы. Она показывает, как компоненты системы взаимодействуют друг с другом.

Для создания диаграммы компонентов используются компоненты, пакеты и отношения зависимости. Каждый компонент может быть настроен с указанием его стереотипа, языка программирования и реализуемых классов.

Пример диаграммы компонентов для системы управления банкоматом включает компоненты "transactions" (библиотека динамической компоновки) и "interface" (интерфейс пользователя). Отношения зависимости между компонентами определяют взаимодействие системы.

14.4. Разработка диаграммы развертывания

Диаграмма развертывания описывает распределение компонентов по узлам. Она используется для моделирования территориально распределенных систем.

Для создания диаграммы развертывания используются процессоры, устройства и соединения. Каждый узел может быть настроен с указанием его стереотипа, характеристик и процессов.

Пример диаграммы развертывания для системы управления банкоматом включает узлы "Банкомат" и "Сервер банка". Соединения между узлами определяют сетевое взаимодействие.

14.5. Генерация программного кода

IBM Rational Rose 2002 позволяет генерировать программный код на различных языках, включая ANSI C++. Процесс генерации включает несколько этапов:

14.5.1. Проверка модели

Перед генерацией кода необходимо проверить модель на наличие ошибок. Это помогает выявить неиспользуемые элементы, несоответствия в именах и другие проблемы.

14.5.2. Создание компонентов

Компоненты создаются для реализации классов. Этот этап выполняется в ходе разработки диаграммы компонентов.

14.5.3. Отображение классов на компоненты

Классы отображаются на компоненты для генерации кода. Это делается через диалоговое окно свойств компонента.

14.5.4. Выбор языка программирования

Для генерации кода выбирается язык программирования, например, ANSI C++. Это настраивается в параметрах модели.

14.5.5. Установка свойств генерации кода

Свойства генерации кода можно настроить для каждого класса или проекта в целом. Это включает выбор стиля кода, комментариев и других параметров.

14.5.6. Генерация кода

Генерация кода выполняется для выбранного класса, компонента или пакета. Созданные файлы можно просмотреть и отредактировать в текстовом редакторе.

Заключение

IBM Rational Rose 2002 является мощным инструментом для разработки и документирования моделей. Он позволяет не только визуализировать систему, но и генерировать программный код, что значительно упрощает процесс разработки.
