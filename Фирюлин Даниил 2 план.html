12.1. Общая характеристика CASE-средства IBM Rational Rose 2002

IBM Rational Rose 2002 — это современное интегрированное средство для анализа, моделирования и разработки программных систем, базирующееся на языке UML. Его основными особенностями являются:

  • Интеграция с MS Visual Studio 6/.NET, включая поддержку прямой и обратной генерации кодов и диаграмм для Visual Basic, Visual C++, Visual J++.
  • Работа с исполняемыми модулями и библиотеками, поддержка технологий Microsoft Transaction Server (MTS) и ActiveX Data Objects (ADO).
  • Поддержка CORBA, компонентной разработки (CBD), языка IDL и DDL.
  • Средства для разработки Java-приложений, включая генерацию классов Java и работу с JAR, CAB и ZIP-файлами.

Эти возможности делают IBM Rational Rose 2002 не только инструментом для моделирования, но и базой данных для современных объектных стандартов.

12.2. Особенности рабочего интерфейса IBM Rational Rose 2002

Рабочий интерфейс Rational Rose 2002 организован таким образом, чтобы обеспечить удобство и функциональность. Основные элементы:

  1. Главное меню — обеспечивает доступ к операциям управления проектами (открытие, сохранение, генерация кода, проверка моделей).
  2. Стандартная панель инструментов — дублирует часто используемые функции главного меню и настраивается через меню Tools → Options.
  3. Окно браузера проекта — отображает иерархическую структуру модели, позволяет находить и редактировать элементы.
  4. Специальная панель инструментов — предназначена для работы с конкретными типами диаграмм, такими как диаграммы классов или последовательностей.
  5. Окно диаграммы — основная рабочая область для визуализации моделей проекта. Одновременно поддерживает отображение нескольких диаграмм.
  6. Окно документации — используется для добавления текстовой информации к элементам модели. Информация преобразуется в комментарии и не влияет на логику кода.
  7. Окно журнала — записывает служебную информацию о действиях пользователя, включая время выполнения операций.

12.3. Назначение операций главного меню

Главное меню в IBM Rational Rose 2002 предоставляет доступ к инструментам работы с системой, включая загрузку, сохранение, редактирование и настройку графического интерфейса. Основные операции меню:

File (Файл)

  • New: создание новой модели.
  • Open: открытие файла модели (.mdl) или подмодели (.ptl).
  • Save / Save As: сохранение модели с возможностью указания имени.
  • Save Log As / Auto Save Log: сохранение журнала в файл.
  • Clear Log: очистка журнала.
  • Load / Save Model Workspace: загрузка/сохранение рабочей области модели.
  • Units: управление версиями модели.
  • Import / Export Model: импорт/экспорт данных модели.
  • Print / Print Setup: печать диаграмм и спецификаций.
  • Edit Path Map: настройка путей к файлам.
  • Exit: выход из программы.

Edit (Редактирование)

  • Undo / Redo: отмена и повтор последнего действия.
  • Cut / Copy / Paste: вырезание, копирование и вставка элементов.
  • Delete / Delete from Model: удаление элементов из диаграммы или модели.
  • Select All: выделение всех элементов диаграммы.
  • Relocate / Reassign: перемещение и замена элементов.
  • Find: поиск элемента по имени.
  • Compartment / Change Info: управление отображением информации об элементах.

View (Вид)

  • Toolbars / Browser / Log: настройка отображения панелей и окон.
  • Zoom / Fit in Window: изменение масштаба диаграмм.
  • Page Breaks / Refresh: разбиение диаграммы на страницы и обновление отображения.
  • Notation Styles: выбор нотаций (Booch, OMT, UML).

Format (Формат)

  • Font Size / Line Color / Fill Color: настройка шрифтов и цветов.
  • Automatic Resize: автоматическое изменение размеров элементов.
  • Stereotype Display: выбор формата отображения стереотипов.
  • Show/Hide Attributes and Operations: управление видимостью элементов модели.

Browse (Обзор)

  • Диаграммы (Use Case, Class, Component и др.): выбор существующих или создание новых диаграмм.
  • Specification / Expand / Parent: управление отображением иерархии модели.

Report (Отчет)

  • Show Usage / Show Instances: информация об элементах модели и их использовании.
  • Show Access Violations: проверка доступа между пакетами.

Query (Запрос)

  • Add Elements / Expand Selected Elements: добавление связанных элементов на диаграмму.
  • Filter Relationships: настройка видимости отношений.

Tools (Инструменты)

Содержит операции, зависящие от установленных расширений, например:

  • Create: создание нового элемента модели.
  • Check Model: проверка модели на ошибки.
  • Model Properties: настройка свойств языка и параметров модели.
